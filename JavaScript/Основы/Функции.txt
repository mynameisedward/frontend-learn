Внутри, функции имеют доступ к глобальным переменным.
Они могут читать, изменять их и т.д


Можно использовать return просто так. 
Это просто приведет к выходу из функции. 


Функция без return и c пустым return возвращает undefined


Функции должны подчиняться правилу: одна функция — одно действие
Функции должны быть короткими и делать только что-то одно. Если это что-то большое, 
    имеет смысл разбить функцию на несколько меньших. 
    Иногда следовать этому правилу непросто, но это определённо хорошее правило.


=========== FUNCTION EXPRESSION ===========

const sayHi = function(name) {
    console.log(`HI ${name}`)
}

Отличия от Function Declaration: (function someFunc() {})
    1. Функция, сделанная с помощью Declaration может быть вызвана раньше чем объявлена
            Движок перед запуском кода находит function declaration и регистрирует их.
            С Function Expression так нельзя
    2. Функция, объявленная с помощью function declaration доступна только внутри блока.
            А объявленная с function expression в любом месте.
