--------------- __proto__ ---------------

1. У каждого объекта в JS есть свойство __proto__
  Это либо какой-то объект либо null
  Если JS не находит поля в исходном объекте, он идет в __proto__ 


2. Как установить __proto__

  - Object.create(x)
    Создает пустой объект с __proto__ равным x

  - Object.setPrototypeOf(obj, x)
    У объекта obj, установит __proto__ равным x

  - obj.__proto__ = x
    Сделает тоже самое что и сверху


3. Как создается объект с __proto__

  Вызывается new Func()
    создается this = { __proto__: Func.prototype }

    возвращается return this


  Примеры:
    const obj1 = new func1()     // obj1.__proto__ === func1.prototype
    const obj2 = new Object()    // obj2.__proto__ === Object.prototype



4. Запись x = {} равнозначана записи x = new Object()


--------------- Определение значения this ---------------

1. План определения значения this:
  0(опциональный) -- Если функция стрелочная, 
                     найти ближайшую обычную функцию, внутри которой находится наша функция
  
  1 -- Найти, где эта обычная функция вызывается

  2 -- В зависимости от того, как эта функция вызывается, определить значение this



2. Как this зависит от способа вызова функции

  - Как обычная функция func()
      this = undefined (в строгом режиме)
      this = globalThis(window в браузере, или global в node.js) в не строгом режиме


  - Как метод obj.foo() или arr[0]()
      this равен объекту слева от точки: this === obj или this === arr


  - С явным указанием this: obj.foo.call(obj2) или func.apply(obj2)
      Когда используем call, bind, apply. То здесь явно задаем this


  - Как конструктор: new func()
      this === { __proto__: func.prototype } - новый объект с определенным полем __proto__
