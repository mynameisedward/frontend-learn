Функциональное программирование - особый стиль написания кода. 
                                  Набор концепций, правил и особенностей при написании кода


4 основных концепции, вытекающих из ФП:
  - Декларативность
  - Чистые функции
  - Функции первого класса. Функции высшего порядка
  - Иммутабельность.(неизменяемость)


----------------- Декларативность -----------------
Декларативность - что хотим получить? описываем результат
Императивность - как хотим получить? описываем действия

код можно писать в одном из этих двух стилей

Под капотом декларативности всегда будет императивный код

Декларативность -- свойство хорошего кода



----------------- Чистые функции -----------------
Критерии чистоты:
- При одних и тех же аргументах, всегда получим один и тот же результат

- Не зависят от внешнего состояния и глобальных переменных
    Потому что если кто-то изменит глобальную переменную, то результат изменится
  Зависят только от того, что приходит в эту функцию аргументом

- Не содержат сайд эффектов
  Например, Если добавим запрос на сервер внутри функции, то функция перестанем стать чистой 
    Потому что мы не можем гарантировать, что бэкенд всегда ответит одним и тем же результатом  
  Также к сайд-эффекту можно отнести изменение глобальных переменных. Когда "изменяем внешний мир"


Хороший пример чистой функции -- reducer из Redux




----------------- Иммутабельность(неизменяемость данных) -----------------

Это аксиома. Данные мутировать нельзя
            Если нужно изменить данные -- правильно создать новый объект/массив и работать именно с НОВЫМ объектом/массивом

В ФП подразумевается, что почти все данные - иммутабельны 

------------
Объекты, массивы, передаются всегда по ссылке, а не по значению
------------




----------------- Функции первого класса. Функции высшего порядка.  -----------------

------ Функции первого класса ------ 

Функции можно:
  - передавать в другие функции как аргументы и вызывать их внутри другой функции
  - вернуть из другой функции
  - можно присвоить функцию переменной
  - можно несколько функций запихнуть в массив


Т.е с функциями можно обращаться как с объектом, + их можно вызвать
И вот когда мы можем вот так работать с функциями, они и называются функциями первого класса


------ Функции высшего порядка ------

Функции высшего порядка - это функции, которые как то работают с другими функциями
Они:
  - либо принимают их аргументом
  - либо возвращают другую функцию

Примеры -- map, filter, reduce










----------------------- Каррирование -----------------------

sum(1, 2, 3, 4, 5)

сделать из того, что сверху, то, что снизу

sum(1)(2)(3)(4)(5)

как сделать такую функцию?

1. создадим функцию curry(fn) -> она будет делать функцию каррированой

  function curry(func) {
  return function curried(...args) {
    if (args.length >= func.length) {
      // вызываем функцию
      return func.apply(this, args);
    } else {
      // возвращаем новую функцию
      return function (...args2) {
        return curried.apply(this, args.concat(args2));
      };
    }
  };
}

const sum = (a, b, c) => a + b + c;
const curriedSum = curry(sum);

console.log(curriedSum(5)(4)(3));


--------------- Chaining(цепочки) ---------------

Это любые цепочки вызовов функций
Примеры:
  curriedSum(5)(4)(3)(2)

  fetch('/api')
    .then(data => data.json())
    .then(data => ({...data, date: Date.now()}))
    .then(console.log)

  new Array(20)
    .map(callback)
    .filter(callback)
    .reduce(callback)
    .map(callback)
