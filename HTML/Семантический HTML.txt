Несемантические элементы — ничего не говорят нам о содержании
    Примеры: <div>, <span>

Семантические элементы — элементы, которые могут рассказать нам о том, что находится внутри
    Примеры: <article>, <form>, <table>


====== Почему семантика важна? =======

• Доступность
    Людям с ограниченными возможностями становится доступней вебсайт
    Незрячие(или частично незрячие люди) используют не веб-браузер, а скринридер, который зачитывает
        содержимое вебсайта и семантическая структура помогает лучше определять, какой сейчас блок
        Например, у скринридера есть функция навигации по заголовкам, что ускоряет знакомство с информацией на сайте
    Таким образом больше людей смогут использовать наш веб-сайт

• Чтобы веб-сайт был выше в поисковиках 
    Известно, что семантическая разметка помогает поисковым ботам, которые определяют какой контент находится на странице.

• Семантика прописана в стандартах
    В стандартах HTML есть семантические теги, которые рекомендуется использовать вместо div и span


====== Основные семантические теги ======

note: старый семантический тег — <p></p> Обозначает параграф

• <article> — тот элемент страницы, который мы можем отделить и смысл не пропадёт.
    Например, комментарий, твит, статья и т.д
    - Особенности: желателен заголовок внутри
    - Ошибки: путают с тегами <section> и <div>

• <section> — смысловой раздел документа. неотделяемый в отличии от <article>
    - Особенности: желателен заголовок внутри
    - Ошибки: путают с тегами <article> и <div>

• <aside> — косвенный, не основной для страницы контент. Обычно это сайдбар сбоку страницы
    - Особенности: может иметь заголовок, может встречаться несколько раз на странице
    - Ошибки: Размечать этим тегом основной контент

• <nav> — навигационный раздел со ссылками на другие страницы или другие части страниц
    Особенности: 
        Используется для основной навигации, а не для всех групп ссылок(1 раз на странице)
        Например, в меню в футере не нужно использовать <nav>
    Ошибки: многие считают, что в <nav> могут быть только ссылки, хотя на деле там может 
        находиться навигация в любом виде

• <header> — вводная часть какого либо раздела, или всего сайта. Обычно содержит подсказки и навигацию
    Особенности: может быть несколько тегов <head> на странице
    Ошибки: использовать только как шапку сайта

• <main> — основное содержание страницы, не повторяющееся на других страницах
    Особенности: Должен быть один на странице
    Ошибки: Вносить в этот тег то, что повторяется на других страницах

• <footer> — заключительная часть раздела или всего сайта
    Особенности: может быть несколько на странице
    Ошибки: использовать только как подвал сайта



====== Есть простые правила для выбора нужных тегов. ======

Получилось найти самый подходящий смысловой тег — использовать его.
Для потоковых контейнеров — <div>.
Для мелких фразовых элементов (слово или фраза) — <span>.
Правило для определения <article>, <section> и <div>:

Можете дать имя разделу и вынести этот раздел на другой сайт? — <article>
Можете дать имя разделу, но вынести на другой сайт не можете? — <section>
Не можете дать имя? Получается что-то наподобие «новости и фотогалерея» или «правая колонка»? — <div>
