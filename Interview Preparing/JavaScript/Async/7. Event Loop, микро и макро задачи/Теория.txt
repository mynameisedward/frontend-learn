Event Loop — это механизм в JavaScript,
    который управляет асинхронными задачами и очередями событий. 
Он позволяет JavaScript работать в однопоточной модели, 
    обрабатывая асинхронные операции, не блокируя основной поток выполнения.


Микротаски — промисы (.then, .catch, .finally), queueMicrotask(), MutationObserver.
    всё, что работает через Promise -- (fetch, async/await) это микрозадачи

Макротаски — setTimeout, setInterval, события DOM (click, input), MessageChannel, requestAnimationFrame.



/////////// Заметка ///////////
fetch это не макротаска
fetch возвращает Promise
///////////////////////////////


------------------ Порядок: ------------------ 
    1. Синхронные операции
    2. Выполнение очереди микрозадач
        Все микротаски выполняются до того, как браузер возьмет следующую макрозадачу   
        Если микротаска добавляет в очередь новую микротаску, она также выполнится в этом цикле
    3. Рендеринг (при необходимости)
        Браузер проверяет, нужно ли выполнить перерисовку страницы
        Если есть изменения DOM или стилей, выполняются: 
            requestAnimationFrame callbacks, пересчет стилей, layout, paint
    4. Одна макрозадача
        Из очереди макрозадач берется одна и выполняется 
        Затем цикл повторяется с шага 2



------------------ Структуры данных: ------------------  

- Call Stack. Стек вызовов
    Содержит функции, которые выполняются в текущий момент

- Web API. 
    Асинхронные операции (setTimeout, обработчики событий, сетевые запросы)
    Всё это передается в Web API.
    После завершения асинхронной операции, callback помещается в соответствующую очередь

- Task Queue / Callback queue 
    Здесь хранятся колбэк функции макрозадач
    Event Loop перемещает задачи из очереди в Call Stack, когда стек пуст

- Microtask queue 
    Отдельная очередь для микрозадач
    Имеет приоритет над Task Queue
    Полностью опустошается после каждой синхронной задачи или макрозадачи



------------------ Микрозадачи ------------------  

- Промисы
    .then(), .catch(), .finally() 

- async/await (продолжение после await)

- queueMicrotask()

- MutationObserver()

- process.nextTick() (Node.js приоритетнее других микротасков)


------------------ Макрозадачи ------------------

- setTimeout(), setInterval()

- события DOM 
    click, input, load

- MessageChannel, postMessage

- setImmediate 
    Node.js

- I/O операции
    Node.js


------------------ requestAnimationFrame ------------------

rAF выполняется после микрозадач, но перед макрозадачами
    непосредственноо перед рендерингом




------------------ Голодание макротасков, Starvation ------------------

Если микрозадачи бесконечно добавляют новые микрозадачи,
    то макрозадачи и рендеринг так никак не выполнятся

Это называется starvation



------------------ async/await и Event Loop ------------------

Код после await превращается в callback внутри .then(),
    т.е попадает в очередь микрозадач




------------------ queueMicrotask ------------------

queueMicrotask() позволяет явно добавить задачу в очередь микрозадач

Полезно, когда нужно отложить выполнение кода, но выполнить его раньше любых макротасок
