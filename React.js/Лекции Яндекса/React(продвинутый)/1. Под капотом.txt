
Весь путь интерфейса разделяется на 2 фазы:

------------- 1 фаза. Rendering и Reconcilation -------------
                  Рендеринг и сравнение

Кроме дерева элементов есть дерево волокон

Для каждого элемента создается Fiber-нода в дереве волокон
    Эта нода хранит состояние элемента и знает об этом реакт-элементе

Fiber - JavaScript объект, у которого есть: 
            stateNode. хранит состояние, знает информацию о пропсах
            child. Дочерний элемет
            return. Родитель
            sibling. Брат


Дерево ли дерево Fiber'ов?
    нет. это Linked List
    По этому можно хорошо навигироваться

Что это за связи?
    - Child. 
        Связь от родителя к ребенку.
        Родительская нода связана только с ОДНИМ дочерним элементом
    - Return(Parent). 
        Связь от ребенка к родителю
        Каждый дочерний fiber знает о родительском fiber
        Все дочерние знают о своем родителе
    - Sibling 
        Братья 
        Дочерние компоненты свзяны между собой

Эта структура позволяет выполнять более быструю навигацию


Всё начинает с Host Root ноды.
    это та самая точка входа в index.js
        createRoot

Это первый рендер(в DOM'е еще ничего не отобразилось):
    1. Впервые построилось дерево React-элементов
    2. Построилось дерево fiber'ов
        Эти файберы связаны. Linked List
        Каждый файбер хранит знание о элементе
        Именно это и позволяет при перерендерах сохранять это состояние
            и это в будущем позволит управлять этой работой


Как всё это показывать пользователю?
    Нужно сделать работу!
    
Информация о том, какую работу нужно сделать хранится в этих файбер-нодах

Работа -- эффекты
эффекты -- запросы данных, подписки и изменения DOM'a
    
Все эффекты связаны. каждый эффект знает о другом эффекте
    Мы берем первый эффект, и выполняем их друг за другом

Все эффекты имеют разные приоритеты

Вся работа приоритезируется и этот список уходит на вторую фазу


Если в какой-то ноде изменилось состояние:
    1. Создается новое дерево. WorkinProgress tree
    2. В новое дерево React клонирует ноды, которые не изменились
    3. Смотрит ноды, в которых нужно сделать работу
    4. Закидывает все эффекты которые нужно сделать в Effect List
    5. Отдаем список в следующую фазу -- Effect List
